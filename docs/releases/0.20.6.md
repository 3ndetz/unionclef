## How to test

```
;goto <x> <y> <z>          — parkour routes, fences, ladders
;followPlayer <nick>        — follow a moving target
;settings enableTrailing    — check trailing state
;settings enableLeap        — check leap state
;settings driftThreshold    — check drift threshold
;settings verboseDebug true — see position/velocity mismatches
```

## Changes

### Tungsten: major A* pathfinding overhaul

Complete rework of cost model, heuristic, and search behavior.
Bot now finds longer parkour routes and falls off paths much less.

**Cost model fixes:**
- Removed fake forwardSpeed penalty that added +15 to ALL nodes —
  A* can now actually distinguish good paths from bad
- Fixed airborne cost accumulation bug (all fall ticks had same
  cost instead of accumulating)
- Yaw penalty tuned to not over-penalize sprint-jumps with turns
- Quantized yaw in closed set (8 directions instead of 360) —
  eliminates redundant exploration of same position with similar angles

**Heuristic stabilization:**
- Removed blockPath_remaining*40 term that changed mid-search when
  waypoint index advanced, breaking A* consistency
- Compensated with increased realTarget distance weight

**Partial path pipeline:**
- Proactive emit: bot starts moving as soon as a decent partial path
  exists, instead of waiting for timeout
- Timeout reduced from 112s to 15s for goto
- Minimum path size reduced from 46 to 15 nodes
- Goto now auto-retries up to 10 times instead of giving up

**Follow improvements:**
- Executor keeps running during recalc (bot doesn't stop to think)
- LEAP mode disabled by default (unreliable close-range movement)
- Trailing disabled by default, toggle via ;settings enableTrailing
- Fixed trailing log spam when disabled

**Search improvements:**
- BlockSpacePathFinder shallow timeout increased from 480ms to 1500ms —
  finds longer parkour routes
- Ladder climbing: 10 ticks per A* node instead of 1 — faster search
- Removed illegal setVelocity(0,0,0) that caused client-server desync

### Shredder: god bridge mode

New bridging mode: `#bridgingMode god` — no-sneak bridging with
jitter click pattern. Use `#bridgingMode slow` to switch back.

### AltoCef: BedWars improvements

- Gear gate: no offensive actions until 48 blocks collected
- Shop priority over looting when low on blocks
- Loot radius scales with gear state
- Fixed shop clicking wrong slots

### AltoCef: Python scripting

Scripts in `scripts/` folder, auto-connect via Py4J.
See `scripts/README.md` for setup.

## Known issues

- Tungsten simulation has collision detection differences with the
  real game (step-up logic), causing position mismatch on fences
  and complex terrain. Drift detection stops the path — investigating
- ;settings verboseDebug true shows position/velocity mismatches
  for diagnosis
