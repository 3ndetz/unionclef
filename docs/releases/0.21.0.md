## Changes

### Tungsten: combat AI

New `;punkPlayer <nick>` command ‚Äî full PvP system with A* approach + combat.

- **TriggerBot** ‚Äî human-like attack clicker with MC key binding cycles
- **WindMouse aiming** ‚Äî anti-cheat safe rotation via raw pixel deltas,
  configurable gravity/wind/maxStep/flickScale
- **SafetySystem** ‚Äî edge detection, anti-fall braking, escape jumps
- **CombatExecutor** (experimental) ‚Äî Agent-simulated combat timeline

Combat settings (`;settings <name> <value>`):
- `combatTriggerBotEnabled` (true) ‚Äî auto-click on target
- `combatRotatesEnabled` (true) ‚Äî auto-rotation toward target
- `combatMovementsEnabled` (false) ‚Äî combat legs (sprint-jump, chase)
- `combatExecutorEnabled` (false) ‚Äî pre-computed attack timeline
- `combatSaverEnabled` (true) ‚Äî edge safety system
- `combatWindMouseGravity` (2.0), `combatWindMouseWind` (0.8),
  `combatWindMouseMaxStep` (4.0), `combatWindMouseWindDist` (15.0),
  `combatWindMouseDoneThreshold` (0.5), `combatWindMouseFlickScale` (3.0)

AltoCef integration: `@punk <nick>` wraps TungstenPunkTask,
replaces old KillPlayerTask across the codebase.

### Tungsten: ViaVersion compatibility

Fixes for older servers accessed through ViaVersion where block
collisions differ from the 1.21 client.

- **`avoidStuckFence`** (default true) ‚Äî `isConnected()` now treats any
  fence/wall/pane/iron bars with an adjacent similar block as connected,
  regardless of client-side state. Blocks paths through invisible
  connection bars. Covers all fence types, walls, glass panes, iron bars.
- **`avoidStuckAnvil`** (default true) ‚Äî anvil collision replaced with
  rotation-invariant center square (0.25‚Äì0.75). Rejects nodes horizontally
  adjacent to anvils. Bot can stand on center top but not edges.
- **Start inside thin blocks** ‚Äî when pathfinding starts inside a fence,
  pane, chain, etc., the start node is moved to the nearest air block
  (cardinal directions first) instead of getting stuck.

### Tungsten: pathfinding fixes

- **Slime NPE fix** ‚Äî starting pathfind on slime no longer crashes
  (null check for `previous` in shouldRemoveNode)
- **Slime bounce condition** ‚Äî inverted to correctly detect falls onto
  slime (previous.y > current.y), expanding search radius for bounce
- **SlimeBounceMove** (experimental) ‚Äî physics-level bounce trajectory,
  but block-space routing through slime still broken (A* can't plan
  "go down to go up" routes ‚Äî documented in TUNGSTEN_SURFACES.md)
- **Chat crash fix** ‚Äî throttle stage change logs, route through mc.execute()
- **Key sticking** ‚Äî TriggerBot reset clears needsRelease flag

## Known issues

- Slime bounce routing: physics work, but pathfinder can't plan
  fall‚Üíbounce‚Üíland routes (architectural A* limitation)
- Ice: friction formula matches vanilla but small drift accumulates
  due to low friction (0.98)
- Scaffolding: physics know about it, but no pathfinding through towers
- `parallelPathfinding` and `closedLoopStrength` are experimental ‚Äî keep disabled
